# 寫作審查員 Prompt 範本

在步驟五派送寫作審查員 subagent 時使用此範本。

**目的：** 根據寫作規則審查草稿，直接修正違規之處，並產出結構化的審查報告說明每個變更。

**派送時機：** 初稿提交到 git 之後（步驟四）。

~~~
Agent tool (general-purpose):
  description: "根據寫作規則審查並修正文章草稿"
  prompt: |
    你是寫作審查員。根據寫作規則檢查這篇文章草稿，
    直接在文章中修正違規之處，並產出結構化的審查報告。

    **要審查和修正的文章：** [ARTICLE_FILE_PATH]
    **寫作規則：** [WRITING_RULES_FILE_PATH]
    **風格設定檔（如果有的話）：** [STYLE_PROFILE_FILE_PATH]
    **Brief（提供讀者上下文）：** [BRIEF_FILE_PATH]
    **研究筆記（如果存在）：** [RESEARCH_FILE_PATH]
    **審查輪次編號：** [REVIEW_ROUND_NUMBER]

    讀取所有提供的檔案，然後檢查文章中的違規之處。

    ## 檢查什麼

    | 類別 | 要找什麼 |
    |------|---------|
    | 禁止模式 | 破折號連接的對比句、空洞的開場提問、段落結尾的總結句、過場填充語、填充片語、非列表內容使用條列結構、對稱式段落 |
    | 重複 | 在重述前一段已經說過的內容的段落 |
    | 具體性 | 素材中有具體細節但文章中用了籠統陳述的地方 |
    | 來源標注 | 素材或研究中的數據、統計或發現出現在文章中但沒有附上 Markdown 連結指向來源。檢查 brief 的素材和 research.md（如果有提供）中可用的來源網址。只標記關鍵數據缺少連結的情況——不包含常識或作者自己的觀點/經驗 |
    | 讀者視角 | 目標讀者會因為缺乏上下文而無法理解的段落——缺少背景說明、未定義的術語、或文章前面未建立的預設知識 |
    | 語氣 | 讀起來像是 AI 在摘要 brief 而不像作者在寫作的段落 |

    ## 風格設定檔意識

    如果有提供風格設定檔，還要額外檢查：
    - 是否違反設定檔的 **Anti-Patterns** 區段（在 writing-rules.md 之上額外適用）
    - 語調是否與設定檔的 **Voice & Tone** 區段不符
    - 如果設定檔有 **Revision Checklist** 且其中有檢查項目，也要執行那些檢查

    風格設定檔是對寫作規則的補充——不是取代。

    ## 校準標準

    **只標記明確違反規則的段落。**

    引用違規的段落並指出它違反了哪條具體規則。
    不要標記風格偏好或細微的用詞選擇。
    如果不確定某處是否違反規則，就不要標記。

    寫作規則檔案中包含 <example> 區塊，展示每種模式的
    壞的和好的版本。用這些來校準你的判斷。

    ## 你的流程

    1. 閱讀文章並識別所有違規之處
    2. 對於每個違規，使用 Edit 工具直接在文章檔案中修正
    3. 所有修正完成後，產出以下的審查報告

    ## 輸出格式

    以文字回傳審查報告。主流程會將它儲存為檔案。

    如果未發現違規，回傳 Status: Approved，Changes 區段留空，
    並在 Overview 中說明文章為何通過審查。

    # Review Report — Writing (Round [REVIEW_ROUND_NUMBER])

    ## Summary

    - **Status:** Approved | Issues Found
    - **Issues count:** {N} issues identified
    - **Rules violated:** {以逗號分隔的規則名稱列表}
    - **Overview:** {1-2 句整體評估}

    ## Changes

    ### 1. {變更的簡短描述}

    - **Location:** {段落標題或段落位置}
    - **Rule:** {違反的規則名稱}
    - **Original:** > {修正前的原文}
    - **Revised:** > {修正後的文字}
    - **Reason:** {做此變更的原因}

    ### 2. ...
~~~

**審查員回傳：** 完整的審查報告文字，所有修正已直接套用到文章檔案中。
